# AURUS Control Panel — Карта XCraft

## [S1] Проблема

Альянсу нужно визуально отображать расположение объектов в игре:
- Планеты и аккаунты игроков по расам
- Альянсовые станции (алстанции) и зоны покрытия бонусов
- ОПС (орбитальные пушечные станции)
- Флот
- Границы альянса

Сейчас координаты хранятся в текстовом поле, нет визуализации. Невозможно планировать расположение алстанций, видеть кластеры игроков, принимать решения о строительстве.

## [S2] Система координат XCraft

### Формат
`X:Y:Z` где:
- **X, Y** — координаты солнечной системы (целые числа)
- **Z** — номер планеты (1-9), 0 = солнце

### Примеры
- `2500:2500:0` — солнце системы альянса
- `2500:2500:1` — планета 1 (ближайшая к солнцу)
- `2500:2500:9` — планета 9 (дальняя)
- `2501:2500:1` — соседняя система, планета 1

### Расстояния
- Между солнечными системами: 1000 единиц
- Между планетами в системе: 100 единиц
- Центр альянса: `2500:2500`

### Формат хранения в БД
Два формата:
- Новый: `[2500:2504:9]` (скобки, двоеточия)
- Старый: `109/22/78` (слеши) — нужно конвертировать

## [S3] Визуализация (HTML5 Canvas)

### Архитектура
```
/map → route → template (canvas + JS)
                    ↓
            fetch /map/api/data → JSON (все объекты)
                    ↓
            Canvas renderer (отрисовка)
```

### Компоненты карты

#### 1. Сетка координат
- Линии по X и Y с шагом 100 (планеты) и 1000 (системы)
- Подписи координат по краям
- Тонкие линии для планет, жирные для систем

#### 2. Солнечные системы
- Круг с номером системы
- 9 точек планет вокруг солнца (по орбите)
- Солнце в центре (жёлтая точка)

#### 3. Объекты на карте

|-----|-------|------|--------|
| Игрок (Жук) | Круг | #27ae60 | 8px |

| Игрок (Тосс) | Круг | #8e44ad | 8px |
| ОПС | Квадрат | #2c3e50 | 6px |

- Полупрозрачный круг вокруг алстанции

- `[2500:2504:9]` → `(2500, 2504, 9)`
- `109/22/78` → `(109, 22, 78)` (старый формат)

## [S5] Инструмент «Планировщик алстанций»

### Назначение
Помочь спланировать расположение алстанций для максимального покрытия бонусами сети.

### Режим работы
1. Показать существующие алстанции (красные ромбы)
2. Показать зоны покрытия (полупрозрачные круги)
3. Показать «дыры» — области без покрытия
4. Предложить оптимальные позиции для новых алстанций

### Алгоритм предложения позиций
1. Взять все существующие алстанции
2. Определить границы альянса (область с игроками)
3. Разбить на сетку с шагом = радиус покрытия
4. Найти точки сетки, не покрыты существующими алстанциями
5. Отсортировать по количеству игроков в радиусе
6. Предложить топ-10 позиций

### Визуализация
- Зелёные пометки — «сюда можно поставить»
- Красные — «уже занято»
- Жёлтые — «частично покрыто»

## [S6] Техническая реализация

### Файлы
```
map/
├── routes/map.py          # Роутер
├── templates/map/index.html  # Страница карты
└── static/js/map.js       # Canvas renderer
```

### Роутер `/map`
- `GET /map` — страница карты
- `GET /map/api/data` — JSON с данными
- `GET /map/api/plan` — предложенные позиции алстанций

### Canvas renderer (map.js)
- Класс `GameMap`:
  - `constructor(canvas)` — инициализация
  - `setData(data)` — загрузка данных
  - `render()` — отрисовка
  - `zoom(delta, x, y)` — зум
  - `pan(dx, dy)` — панорамирование
  - `findObject(x, y)` — поиск объекта под курсором
  - `centerOn(x, y)` — центрирование
  - `exportPNG()` — экспорт

### Производительность
- Virtual rendering — отрисовывать только видимые объекты
- Кэширование при статичном виде
- requestAnimationFrame для плавности
